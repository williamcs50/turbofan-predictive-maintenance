## Where v0.2 stands

The starting point isn't a fresh feature. It's a diagnosis that closed.

v0.1 came back precision 0.00, recall 0.00, RMSE 41.72. Both heads flat. That looked like a broken model for weeks. It wasn't. It was the model correctly reporting that there was no learnable signal in the v1 data. An honest instrument reading a dead channel. The fix was the data, not the model.

The v2 generator puts per-channel signal into all five failure modes. Short column names (`vibration`, `T50`, `P30`, `Nf`, `fuel_flow`) let `preprocess.py` adapt with no changes. Exclusion-based column detection in `EngineDataset` handles all five channels, `num_features=5`, batch shape `(64, 50, 5)`. The learnability gate, a simple linear model on windowed means, passed all five modes, per-mode recall 0.80–0.84, F1 0.87.

The Transformer retrained on v2 (`v0.2.0`) came back:

| Metric              | v0.1  | v0.2.0 |
|---------------------|-------|--------|
| Anomaly precision   | 0.00  | 0.96   |
| Anomaly recall      | 0.00  | 0.80   |
| RUL RMSE (cycles)   | 41.72 | 8.92   |

Same architecture, same loop. Swap in data with signal and it learns. That's the diagnostic chain closing: concluded the data was the problem, corrected the data, first fair test confirmed it.

**Two findings carried out of v0.2.0 that this cycle acts on:**

1. **The model is miscalibrated for the domain.** Precision 0.96 / recall 0.80 means it would rather miss a failure than raise a false alarm. On a turbofan that's backwards. A missed failure costs far more than a false alarm. Traced to the class weights. This is what v0.2.1 fixes.
2. **The gate and the Transformer are tied** (both ≈ F1 0.87 on classification). Not a failure. A finding. The degradation here is a clean monotonic ramp, and a windowed mean captures a ramp fine, so attention buys little on *concentrated*-SNR data. The Transformer is built for the *distributed*-SNR regime, where signal is spread thin across channels and no single channel suffices. That regime is untested. The distributed-SNR experiment runs the test instead of telling the story.

## The project: four pieces, one thing

This is one multi-day project, not four errands. The pieces are sequenced; reaching for the later ones early just fragments the thread.

### 1. v0.2.1: cost-weighted operating point
The core of the cycle. Choose a scoring metric that matches the domain asymmetry: Fβ weighted toward recall, or recall-at-a-precision-floor (pick whichever fits and defend it). Run the threshold sweep. Choose the operating point: how many false alarms am I willing to eat to stop missing real failures. That choice is a judgment call argued from the domain, not a number the code hands me. It can legitimately take the afternoon, and if it does, that's the work going well.

- **Tags under:** `v0.2.1` (patch: correcting how the existing model decides, not changing what it is)
- **Done when:** metric chosen and implemented, sweep run, operating point selected and defensible, tagged.

### 2. Cleanup
Rides under the same `v0.2.1` tag. Class weights wire in as part of the operating-point change, so that item comes along for free. Small items if there's daylight: PYTHONPATH, the checkpoint footgun. **Also park, don't act:** check what the `feat/v0.3-simulator-redesign` branch (on old commit `b577f09`) actually is: stale or a real start.

- **Tags under:** `v0.2.1`
- **Done when:** folded into the v0.2.1 change; the v0.3-branch question is resolved or explicitly parked in the log.

### 3. Distributed-SNR experiment
Flip `DISTRIBUTED=True`, spread the signal thin across channels so no single channel suffices, retrain. Tests the hypothesis that attention earns its cost in the distributed regime. This experiment also decides whether v0.3 is worth doing: if the Transformer clearly earns its keep here, the current architecture holds and a simulator redesign is hard to justify; if it doesn't, that's the evidence that the simulator and architecture need rethinking, and v0.3 gets a reason to exist.

- **Tags under:** TBD by result (informs whether v0.3 exists)
- **Later in the week, not Tuesday.**
- **Done when:** flag flipped, retrained, gap measured, result written down either way.

### 4. Case study page
Walks through the v0.2 arc by version: v0.2.0 baseline → v0.2.1 cost-weighted fix → the distributed-SNR finding. Docs and site. Carries no code version.

- **Tags under:** none (docs/site)
- **End of week.**
- **Done when:** the arc reads the way the tags read.
